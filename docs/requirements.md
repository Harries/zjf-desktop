# zjf.ai Desktop 需求整理

## 1. 产品定位

zjf.ai Desktop 是 zjf.ai 的桌面客户端。用户只需要配置自己的 zjf.ai API Token，就可以在本地桌面端完成图片上传、浏览、复制链接、删除和基础管理，不需要频繁打开网页控制台。

当前公开信息显示，zjf.ai 是面向开发者的图床服务，支持图片上传、智能压缩、格式转换、私密存储、API 调用、Markdown 外链、批量上传、拖拽上传等能力。公开 API 文档已覆盖通过 Token 上传图片、获取图片列表、删除图片等基础能力。

## 2. 目标用户

- 博客、文档、知识库作者：希望快速上传图片并复制 Markdown/URL。
- 开发者：希望通过桌面工具管理图床资源，减少网页登录操作。
- 设计、运营、内容创作者：希望批量上传、查找和清理图片资源。

## 3. 核心价值

- 低配置成本：首次启动只需要填写 zjf.ai Token。
- 高效率上传：支持拖拽、粘贴、文件选择、批量上传。
- 本地化管理：在桌面端查看、搜索、复制和删除远端图片。
- 适配写作流：快速复制原始链接、Markdown、HTML 或 BBCode。

## 4. MVP 范围

### 4.1 首次配置

- 用户打开应用后，如果没有 Token，展示 Token 配置页。
- 支持输入、保存、修改和清除 Token。
- Token 保存在系统安全存储中，优先使用 Keychain、Credential Manager 或系统等价能力。
- 配置成功后自动拉取图片列表，用一次真实 API 请求验证 Token 是否可用。

### 4.2 图片上传

- 支持从本地选择图片上传。
- 支持拖拽图片到窗口上传。
- 支持从剪贴板粘贴图片上传。
- 支持批量上传，并展示每个文件的上传状态。
- 上传成功后展示缩略图、文件名、大小、图片尺寸、链接和上传时间。
- 上传失败时展示明确错误，例如 Token 无效、网络失败、文件类型不支持、文件过大。

### 4.3 图片列表

- 启动应用或刷新时，从 zjf.ai API 获取图片列表。
- 支持网格视图，默认按上传时间倒序。
- 支持查看缩略图、文件名、大小、格式、上传时间。
- 支持手动刷新。
- 支持本地搜索，搜索字段至少包括文件名和链接。
- 支持空状态、加载状态、错误状态。

### 4.4 图片操作

- 复制图片 URL。
- 复制 Markdown 格式。
- 复制 HTML img 标签。
- 在默认浏览器打开图片。
- 删除远端图片，删除前二次确认。
- 删除成功后从本地列表移除。

### 4.5 设置

- 查看当前 Token 配置状态，不明文展示完整 Token。
- 修改 Token。
- 清除 Token 并回到首次配置页。
- 配置复制格式偏好，例如默认复制 URL 或 Markdown。
- 配置上传后是否自动复制链接。

## 5. 非 MVP / 后续能力

- 文件夹、标签、收藏、批量重命名等高级整理能力。
- 按颜色、尺寸、格式、上传日期筛选。
- 本地缓存缩略图，支持离线查看最近图片。
- 系统菜单栏快速上传。
- 全局快捷键上传剪贴板图片。
- 上传前压缩、格式转换、质量参数设置。
- 图片详情页展示 EXIF、尺寸、访问链接、隐私状态。
- 多账号 / 多 Token 管理。
- 私有图片 Signed URL 管理。
  - 注意：公开文档中 Signed URL 当前标注为“暂时只支持浏览器请求，不能通过 API 直接获取”，所以桌面端不能把它作为首版强依赖。

## 6. 页面与交互

### 6.1 主页面

- 顶部工具栏：上传、粘贴上传、刷新、搜索、设置。
- 主区域：图片网格。
- 右侧或弹窗：图片详情与复制操作。
- 底部或队列区域：上传进度和失败重试。

### 6.2 Token 配置页

- Token 输入框。
- 保存并验证按钮。
- 前往 zjf.ai 获取 Token 的链接。
- Token 验证失败时给出原因和重试入口。

### 6.3 上传队列

- 展示文件名、进度、状态。
- 支持失败重试。
- 支持清除已完成项。

## 7. API 依赖

基于公开文档，MVP 至少依赖以下能力：

- 上传图片：`POST /api/upload`
- 获取图片列表：`GET /api/uploads?pageSize=100`
- 删除图片：`DELETE /api/uploads/:id`

所有请求都需要在请求头中携带 Token，例如：

```http
Authorization: Bearer <token>
```

需要在开发阶段进一步确认：

- 图片列表接口的分页、排序、搜索能力。
- 上传接口的文件大小限制和支持格式。
- 返回字段是否包含缩略图、原图 URL、图片尺寸、文件大小、创建时间。
- 删除接口需要的图片 id 字段来源。
- API 错误码和错误消息格式。

## 8. 本地数据与安全

- Token 不写入普通明文配置文件。
- 本地可以缓存图片列表元数据，但不缓存 Token。
- 缩略图缓存可作为后续优化，需提供清理缓存入口。
- 日志中禁止输出 Token、完整 Authorization 头。
- 网络请求失败应避免泄漏敏感信息到错误提示。

## 9. 验收标准

### Token 配置

- 新用户首次启动能看到 Token 配置页。
- 输入有效 Token 后进入主页面并成功加载图片列表。
- 输入无效 Token 时不能进入主页面，并提示验证失败。

### 上传

- 拖拽单张图片可成功上传。
- 选择多张图片可并行或串行上传，并展示每张状态。
- 上传成功后图片出现在列表顶部。
- 上传后可一键复制默认格式链接。

### 管理

- 刷新列表能获取远端最新图片。
- 搜索文件名可以过滤列表。
- 删除图片前出现确认。
- 删除成功后列表同步更新。

### 安全

- 重启应用后无需重新输入 Token。
- 设置页不会明文展示完整 Token。
- 应用日志中搜不到 Token 原文。

## 10. 推荐开发优先级

1. 项目脚手架与桌面壳。
2. Token 安全存储与验证。
3. zjf.ai API 客户端封装。
4. 图片列表与基础刷新。
5. 上传队列、拖拽上传、粘贴上传。
6. 复制链接与删除图片。
7. 设置页与错误状态完善。
8. 本地缓存、批量操作和快捷入口。

## 11. 主要风险

- API 文档未明确分页、字段完整性和限制，可能影响列表和详情设计。
- 私有图片 Signed URL 暂不能通过 Token API 直接获取，首版应避免承诺完整私有图管理。
- 如果 Token 只适合服务端调用，需要确认桌面端直接请求是否存在 CORS、鉴权或风控限制。
- 批量上传需要处理速率限制、失败重试和部分成功状态。

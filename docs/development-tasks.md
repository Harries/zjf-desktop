# ZJF Desktop 开发任务文档

## 1. 任务拆分原则

本任务文档基于 `docs/requirements.md`、`docs/technical-architecture.md` 和 `prototype/index.html` 拆分。首版目标是完成一个可用的 zjf.ai 桌面客户端：用户配置 Token 后，可以上传、浏览、搜索、复制链接、删除图片，并管理基础设置。

任务按阶段推进，每个阶段都应能独立验收。建议先完成 MVP 主链路，再补充优化和跨平台发布。

## 2. 阶段总览

| 阶段 | 名称 | 目标 |
| --- | --- | --- |
| P0 | 项目初始化 | 建立 Tauri + React + TypeScript 基础工程 |
| P1 | 桌面基础能力 | 实现 Token 安全存储、设置存储、命令桥接 |
| P2 | zjf.ai API | 封装上传、列表、删除、错误映射 |
| P3 | 页面骨架 | 按原型完成主要页面和路由 |
| P4 | 图片列表 | 实现图库、搜索、详情、复制链接 |
| P5 | 上传能力 | 实现文件选择、拖拽、粘贴、上传队列 |
| P6 | 删除与设置 | 实现删除确认、Token 管理、偏好配置 |
| P7 | 验收与发布 | 完成测试、日志脱敏、构建和发布准备 |

## 3. P0 项目初始化

### TASK-001 初始化 Tauri 项目 - Done

目标：建立可运行的桌面应用骨架。

工作内容：

- 初始化 Tauri 2 + React + TypeScript + Vite。
- 配置 `src/` 和 `src-tauri/` 目录。
- 配置开发命令和构建命令。
- 设置应用名称、窗口尺寸、应用图标占位。

产出：

- `package.json`
- `vite.config.ts`
- `src-tauri/tauri.conf.json`
- `src-tauri/Cargo.toml`
- 可运行的默认桌面窗口。

验收标准：

- 执行开发命令可以打开桌面窗口。
- 前端热更新可用。
- Tauri build 命令至少能进入构建流程。

### TASK-002 建立代码规范 - Done

目标：保证后续代码风格一致。

工作内容：

- 配置 TypeScript strict。
- 配置 ESLint。
- 配置 Prettier 或等价格式化工具。
- 配置 Rust fmt 和 clippy 检查命令。

产出：

- 前端 lint / format 命令。
- Rust fmt / clippy 命令。

验收标准：

- 空项目通过 lint。
- 空项目通过 typecheck。
- Rust 代码通过 fmt。

### TASK-003 建立基础目录结构 - Done

目标：按照架构文档创建模块边界。

工作内容：

- 创建前端目录：`app`、`pages`、`components`、`stores`、`api`、`types`。
- 创建 Tauri 后端目录：`commands`、`services`、`models`。
- 添加占位导出文件。

验收标准：

- 目录结构符合技术架构文档。
- 前后端模块可以正常编译。

## 4. P1 桌面基础能力

### TASK-004 实现 Tauri 命令桥接封装 - Done

目标：前端通过统一入口调用桌面后端能力。

工作内容：

- 创建 `src/api/desktop-commands.ts`。
- 封装 `invoke` 调用。
- 统一处理后端错误对象。
- 定义基础命令类型。

验收标准：

- 前端不直接散落调用 `invoke`。
- 命令调用失败时返回统一错误结构。

### TASK-005 实现 TokenStore - Done

目标：Token 存放在系统安全存储中。

工作内容：

- 在 Rust 后端实现 `token_store` 服务。
- 支持保存 Token、读取 Token、清除 Token、返回脱敏 Token。
- 避免将 Token 写入普通配置文件。

建议命令：

- `save_token(token)`
- `get_token_status()`
- `clear_token()`

验收标准：

- 保存 Token 后重启应用仍能读取 Token 状态。
- 清除 Token 后状态为未配置。
- 设置页只展示脱敏 Token。

### TASK-006 实现 SettingsStore - Done

目标：保存非敏感偏好设置。

工作内容：

- 保存默认复制格式。
- 保存上传后自动复制开关。
- 保存缩略图缓存开关。
- 提供默认值。

验收标准：

- 设置修改后重启仍然生效。
- Token 不出现在设置存储文件中。

### TASK-007 实现剪贴板和外部打开命令 - Done

目标：支持复制链接和在默认浏览器打开图片。

工作内容：

- 实现写入剪贴板命令。
- 实现打开外部 URL 命令。
- 处理失败错误。

验收标准：

- 前端调用后系统剪贴板内容正确。
- URL 可以在默认浏览器打开。
- 失败时前端收到可展示错误。

## 5. P2 zjf.ai API

### TASK-008 实现 API 数据模型 - Done

目标：建立内部稳定模型，隔离 zjf.ai 原始响应。

工作内容：

- 定义 `RemoteImage`。
- 定义 `UploadTask`。
- 定义 `AppError`。
- 定义 zjf.ai 原始响应结构。
- 编写 API 响应到内部模型的转换函数。

验收标准：

- 前端不直接依赖原始 API 字段。
- 字段缺失时有合理默认值或可选值。

### TASK-009 实现 ZjfApiClient - Done

目标：封装 zjf.ai 远端接口。

工作内容：

- 使用 `reqwest` 创建 API Client。
- 统一添加 Authorization 请求头。
- 实现超时配置。
- 实现错误映射。

接口：

- `validate_token`
- `list_images`
- `upload_image`
- `delete_image`

验收标准：

- Token 无效映射为 `TOKEN_INVALID`。
- 网络失败映射为 `NETWORK_ERROR`。
- 文件过大映射为 `FILE_TOO_LARGE`。
- 不向前端返回 Token 或 Authorization。

### TASK-010 实现 Token 验证命令 - Done

目标：Token 配置页可以验证 Token。

工作内容：

- 前端提交 Token。
- 后端使用 Token 调用列表接口或专用验证接口。
- 验证成功后保存 Token。
- 验证失败时返回错误。

验收标准：

- 有效 Token 进入图库。
- 无效 Token 停留在配置页。
- 错误提示明确。

### TASK-011 实现图片列表命令 - Done

目标：前端可以获取远端图片列表。

工作内容：

- 后端从安全存储读取 Token。
- 调用 `GET /api/uploads?pageSize=100`。
- 返回内部 `RemoteImage[]`。

验收标准：

- 有 Token 时可以拉取列表。
- 无 Token 时返回 `TOKEN_INVALID` 或未配置错误。
- 列表按上传时间倒序展示，若 API 已排序则保留。

### TASK-012 实现图片上传命令 - Done

目标：支持上传本地图片文件。

工作内容：

- 接收文件路径或文件数据。
- 校验文件是否存在。
- 调用 `POST /api/upload`。
- 返回上传成功后的 `RemoteImage`。

验收标准：

- 单张图片可以上传成功。
- 上传失败返回可展示错误。
- 上传成功后不泄漏本地完整路径。

### TASK-013 实现图片删除命令 - Done

目标：支持删除远端图片。

工作内容：

- 接收图片 id。
- 后端读取 Token。
- 调用 `DELETE /api/uploads/:id`。
- 返回删除结果。

验收标准：

- 删除成功后前端列表移除该图片。
- 删除失败时图片仍保留。
- 无 Token 或权限不足时提示重新配置。

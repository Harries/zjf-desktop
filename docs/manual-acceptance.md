# zjf.ai Desktop 手动验收清单

验收前准备：

- 一枚有效的 zjf.ai API Token。
- 一枚无效 Token，用于验证错误提示。
- 至少 3 张本地图片，包含 PNG/JPG/WebP 任意格式。
- 一张复制到剪贴板的截图图片。

验收步骤：

- 首次启动应进入 Token 配置页。
- 输入无效 Token，应显示清晰错误，不展示内部堆栈或完整 Token。
- 输入有效 Token，应进入图库，并在侧边栏展示脱敏 Token。
- 图库刷新按钮应重新拉取图片列表。
- 搜索文件名或 URL，应实时过滤图片；清空后恢复全量列表。
- 文件选择上传一张或多张图片，上传队列应出现任务，成功后图库顶部出现新图片。
- 拖拽图片到图库 Dropzone，应进入上传队列并上传。
- 粘贴剪贴板截图，应进入上传队列并上传；粘贴纯文本不应触发上传。
- 上传失败任务应保留错误原因，并能在上传队列页重试。
- 图片详情页 URL、Markdown、HTML 三种复制格式应正确。
- 删除图片必须先弹确认框，取消不调用删除，确认后图库不再展示该图片。
- 设置页切换默认复制格式、自动复制、缓存开关后，重启应用应保持设置。
- 清除 Token 后应返回 Token 配置页，且无法继续调用图片接口。

验收记录：

- 当前状态：待使用真实 zjf.ai Token 执行。
- 阻塞原因：本地开发环境未提供可用于真实上传、删除的 Token。
